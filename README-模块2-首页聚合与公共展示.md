# 首页聚合与公共展示

本模块负责校园二手平台的门户首页，承担用户进入小程序后的内容聚合、公共信息展示和核心功能跳转。当前实现覆盖首页轮播、快捷入口、校园天气、校园头条、个性化推荐、书籍/物品数据加载、跑腿入口和详情跳转。

## 三、首页结构

首页自上而下包含：

1. 搜索栏：输入关键词后跳转结果页。
2. 轮播图：书籍、物品、跑腿三个公共入口。
3. 快捷入口：找书籍、找物品、找跑腿、去发布。
4. 校园天气：展示交易友好提示。
5. 分类快捷 Chip：数码、日用、服饰、跑腿。
6. 校园头条：交易提醒和服务动态。
7. 今日推荐：综合推荐内容。
8. 书籍推荐：书籍类内容。
9. 物品推荐：普通物品类内容。

首页不展示地图，也不展示校区选择卡片。校区只在后台用于推荐排序。

## 四、数据来源

首页推荐数据统一读取 Bmob 云数据库 `Item` 表。

查询基础条件：

```js
status = 'ON_SALE'
order('-createdAt')
```

不同区域的数据来源：

| 区域 | 查询类型 | 展示数量 |
|------|----------|----------|
| 今日推荐 | `type = all` | 先取 24 条候选，再个性化排序取 6 条 |
| 书籍推荐 | `type = book` | 先取 14 条候选，再个性化排序取 4 条 |
| 物品推荐 | `type = goods` | 先取 14 条候选，再个性化排序取 4 条 |

跑腿内容通过结果页展示，首页入口跳转时携带：

```text
type=parttime&category=跑腿
```

说明：内部仍兼容旧参数值 `parttime`，但页面展示和业务语义统一为“跑腿”。

## 五、个性化推荐规则

当前个性化推荐采用轻量级前端排序，不需要新增后端服务。

推荐分数主要由三部分组成：

| 因子 | 说明 |
|------|------|
| 校区匹配 | 登录后定位识别东/西校区，同校区内容优先 |
| 用户偏好 | 用户点击轮播、快捷入口、分类、商品卡片或搜索时记录类型/分类偏好 |
| 发布时间 | 越新的在售内容权重越高 |

排序流程：

```text
读取候选 Item
  ↓
识别每条内容的校区、类型、分类
  ↓
根据校区匹配、偏好分、发布时间计算 score
  ↓
按 score 降序排列
  ↓
截取首页展示数量
```

偏好保存在本地缓存：

```text
homePersonalPreference
```

校区偏好保存在本地缓存：

```text
homeCampusPreference
```

如果旧数据没有 `campus` 字段，会从标题、描述、取件地点、送达地点中识别“东区 / 西区”等关键词作为兜底。

## 六、登录后定位

登录成功后，`pages/login/login.js` 调用：

```js
campusPreference.detectCampusByLocation()
```

定位流程：

1. 调用 `wx.getSetting()` 检查是否已有定位权限。
2. 首次授权时调用 `wx.authorize()`，弹出定位授权框。
3. 如果用户曾拒绝授权，则弹窗引导进入设置页。
4. 授权后调用 `wx.getLocation()` 获取当前位置。
5. 根据当前位置与东/西校区中心点距离，判断推荐校区。
6. 写入本地缓存，用于首页推荐排序。

首页不会显示地图，也不会展示具体定位点。

当前东/西校区经纬度位于：

```js
utils/campusPreference.js
```

后续拿到学校真实坐标后，只需替换 `CAMPUS_POINTS` 即可。

## 七、跑腿分类

跑腿结果页使用细分类：

```text
全部跑腿 / 书籍跑腿 / 数码跑腿 / 日用跑腿 / 服饰跑腿 / 其他跑腿
```

发布跑腿时新增 `errandCategory` 字段，用于保存跑腿类型。

旧跑腿数据若没有 `errandCategory`，系统会根据标题、说明、取件地点、送达地点进行关键词推断：

| 推断类型 | 关键词示例 |
|----------|------------|
| 书籍跑腿 | 书、教材、资料、试卷、图书 |
| 数码跑腿 | 手机、电脑、耳机、平板、充电器 |
| 日用跑腿 | 食堂、快递、水、纸、药、洗护 |
| 服饰跑腿 | 衣、鞋、帽、外套、包 |
| 其他跑腿 | 无法命中以上规则时兜底 |


```text

```

```text

```


```text

```

## 九、自测流程

1. 清除授权数据后重新登录，确认登录后弹出定位授权。
2. 授权定位后进入首页，确认首页不展示地图、不展示校区选择卡片。
3. 发布一条书籍商品，确认首页书籍推荐可展示。
4. 发布一条普通物品，确认首页物品推荐可展示。
5. 发布一条跑腿任务，选择跑腿类型，例如“书籍跑腿”。
6. 首页点击“找跑腿”，确认进入跑腿结果页。
7. 跑腿结果页检查分类 Chip：全部跑腿、书籍跑腿、数码跑腿、日用跑腿、服饰跑腿、其他跑腿。
8. 点击首页分类、轮播或商品卡片后返回首页，确认推荐排序会逐步偏向最近点击的类别。
9. 点击推荐卡片，确认进入详情页。
10. 下拉刷新首页，确认天气和推荐内容正常刷新。
