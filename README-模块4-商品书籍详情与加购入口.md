# 模块 ④ · 商品/书籍详情与加购入口（约 12%）

本模块负责校园二手平台中的 **单商品交互链路**：用户从首页或购物车进入商品详情页，查看商品/书籍信息，并完成加入购物车、立即购买、联系卖家等操作。

> 说明：当前项目中没有 `pages/movie-detail/`、`pages/thing-detail/` 两个目录，实际实现统一收敛在 `pages/itemDetail/`。书籍属于物品分类之一，通过 `Item.category = 书籍` 区分。

## 三、页面入口

### 3.1 首页进入详情

首页商品列表点击卡片：

```js
wx.navigateTo({ url: `/pages/itemDetail/itemDetail?id=${id}` })
```

对应文件：`pages/index/index.js`

### 3.2 购物车进入详情

购物车点击商品：

```js
wx.navigateTo({ url: `/pages/itemDetail/itemDetail?id=${id}` })
```

对应文件：`pages/my/myCart/myCart.js`

## 四、数据表设计

### 4.1 `Item` 表

详情页主要读取 `Item` 表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `objectId` | String | 商品唯一 ID |
| `sellerId` | String / Pointer | 卖家用户 ID |
| `postType` | String | `goods` 普通商品，`errand` 跑腿 |
| `title` | String | 商品/书籍标题 |
| `price` | Number | 商品价格 |
| `category` | String | 分类，如 `书籍`、`数码`、`日用` |
| `description` | String | 商品描述 |
| `coverImage` | String | 封面图链接 |
| `images` | String / Array | 多图数据 |
| `status` | String | `ON_SALE`、`IN_TRADING`、`SOLD_OUT`、`OFFLINE` |

### 4.2 `Cart` 表

点击“加入购物车”时写入。

| 字段 | 类型 | 说明 |
|------|------|------|
| `userId` | Pointer → `_User` | 当前用户 |
| `itemId` | Pointer → `Item` | 商品 |
| `sellerId` | String | 卖家 ID，便于后续扩展 |
| `itemTitle` | String | 商品标题冗余 |
| `itemImage` | String | 商品封面冗余 |
| `price` | Number | 商品价格冗余 |
| `createdAt` | Date | 创建时间 |

### 4.3 `Order` 表

点击“立即购买”时写入。

| 字段 | 类型 | 说明 |
|------|------|------|
| `buyerId` | String | 买家 ID |
| `sellerId` | String | 卖家 ID |
| `itemId` | String | 商品 ID |
| `status` | String | 初始为 `PENDING_CONFIRM` |
| `itemTitle` | String | 商品标题冗余 |
| `itemImage` | String | 商品封面冗余 |
| `price` | Number | 商品价格冗余 |
| `createdAt` | Date | 创建时间 |

## 五、交互流程

### 5.1 加入购物车

1. 用户进入商品详情页。
2. 点击底部“加入购物车”。
3. 调用 `auth.guardCampusAction()` 校验是否登录、是否完成校园认证。
4. 重新读取商品，确认商品仍存在且状态为 `ON_SALE`。
5. 判断当前用户不能加入自己发布的商品。
6. 查询 `Cart` 表，防止重复加入。
7. 写入 `Cart` 表。
8. 提示“已加入购物车”。

### 5.2 立即购买

1. 用户进入商品详情页。
2. 点击底部“立即购买”。
3. 校验登录、校园认证、商品状态、商品归属。
4. 查询 `Order` 表，避免重复创建待处理订单。
5. 创建 `Order` 记录，状态为 `PENDING_CONFIRM`。
6. 跳转到“我买到的”订单列表。

## 六、身份与权限规则

| 场景 | 规则 |
|------|------|
| 未登录 | 不能加购/购买，跳转登录页 |
| 未校园认证 | 不能加购/购买，跳转校园认证页 |
| 查看商品 | 游客也可查看详情 |
| 自己发布的商品 | 不显示买家操作栏，不能购买自己的商品 |
| 商品已下架/已结束 | 不允许加购或购买 |
| 跑腿任务 | 不走商品加购/购买流程，使用接单逻辑 |

## 八、自测流程

1. 登录账号并完成校园认证。
2. 使用另一个账号发布一条商品，分类可选 `书籍`。
3. 回到首页，点击该商品进入详情页。
4. 检查标题、图片、价格、分类、描述是否正确展示。
5. 点击“加入购物车”，确认提示成功。
6. 进入“我的 → 购物车”，确认商品出现在列表中。
7. 再次点击“加入购物车”，确认提示已在购物车中。
8. 点击“立即购买”，确认创建订单并跳转到“我买到的”。
9. 进入“我的 → 我买到的”，确认订单状态为“待确认”。
10. 使用商品发布者账号进入详情页，确认不会出现买家操作栏。

## 九、答辩说明口径

本模块约占项目工作量 **12%**，主要解决单个商品从“查看”到“表达购买意向”的完整交互。  
详情页不仅展示商品/书籍信息，还负责关键操作的身份校验，包括登录、校园认证、商品状态、商品归属校验。  
“加入购物车”写入 `Cart` 表，“立即购买”写入 `Order` 表，为后续购物车、订单管理、卖家确认交易等模块提供数据基础。
