# 信用分机制实现说明

## 实现范围

本次已补齐校园二手交易平台的信用分机制，覆盖用户信用初始化、信用门禁、订单完成加分、异常扣分、信用流水、订单互评、低信用展示降权、申诉/举报裁决扣分，以及云端定时巡检。

## 小程序端改动

### 信用分核心服务

修改 `utils/credit.js`，统一信用分相关逻辑：

- 新用户默认信用分为 `100`。
- 信用分最低不低于 `0`。
- 低于 `60` 分时限制发布、拍下、确认付款、接单等高风险操作。
- 信用分低于冻结阈值时自动标记账号 `status = frozen`。
- 订单完成时买卖双方各 `+2`。
- 卖家超时未响应默认 `-5`。
- 主动/恶意取消订单默认 `-10`。
- 跑腿违约默认 `-10`。
- 虚假信息默认 `-20`。
- 违规发布默认 `-15`。
- 纠纷责任方默认 `-15`，也支持管理端传入指定扣分值。
- 每次信用变更都会写入 `CreditRecord` 信用流水。

### 低信用展示降权

修改 `utils/publish.js` 和 `pages/index/index.js`：

- 商品列表查询时读取卖家信用分。
- 卖家信用分在 `61~80` 区间时，对首页和搜索结果排序进行降权。
- 低于 `60` 分的用户已无法发布新内容。

### 订单互评

新增：

- `utils/review.js`
- `pages/orderReview/orderReview.js`
- `pages/orderReview/orderReview.wxml`
- `pages/orderReview/orderReview.wxss`
- `pages/orderReview/orderReview.json`

并在 `app.json` 注册页面。

- 评价记录写入 `OrderReview` 表。
- 买家评价后回写 `Order.buyerReviewed = true`。
- 卖家评价后回写 `Order.sellerReviewed = true`。
- 订单详情页展示已有评价。

### adminDisputeDecide 云函数

新增 `cloudfunctions/adminDisputeDecide/index.js`。

功能：

- 根据 `disputeId` 或 `caseKey` 查找案卷。
- 关闭申诉/举报案卷。
- 根据责任方扣信用分。
- 写入 `CreditRecord`。
- 推进订单状态为 `COMPLETED` 或 `CANCELLED`。
- 解冻订单 `frozen = false`。
- 同步商品状态。
- 写入 `UserNotice` 结案通知。

### creditSweep 云函数

新增 `cloudfunctions/creditSweep/index.js`。

功能：

- 定时扫描 `IN_TRADING` 订单。
- 如果 `payExpireAt` 超时，自动取消订单。
- 释放商品锁定并恢复在售。
- 对卖家执行超时扣分。
- 定时扫描长期 `SHIPPED` 订单并自动完成。
- 自动完成时触发双方信用加分。

字段已同步为：

- `timeoutCreditSet`
- `creditSettledAt` 日期对象格式

### 定时任务

`creditSweep` 已配置为定时任务。

执行命令使用 Bmob 6 位定时规则：

```text
0 */5 * * * *
```

如套餐限制不允许 5 分钟一次，可改为每小时：

```text
0 0 * * * *
```

## 后续本地闭环自测

建议在微信开发者工具中继续确认：

1. 重新编译小程序。
2. 完成一笔订单，确认买卖双方信用分各 `+2`。
3. 查看 `CreditRecord` 是否新增 `order_complete` 流水。
4. 订单完成后进入订单详情，确认出现“评价对方”。
5. 提交评价后，确认 `OrderReview` 新增记录。
6. 将测试账号信用分调到 `< 60`，确认无法发布、拍下、付款、接单。
7. 将卖家信用分调到 `61~80`，确认首页/搜索展示顺序被压低。
8. 申诉裁决后，确认消息页能看到结案通知，信用明细能看到 `纠纷裁决` 扣分。
