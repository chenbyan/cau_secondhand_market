yue'du# 数据存储与配置说明

本文档说明本项目的 **数据存在哪里**、**配置文件改什么**、以及常见误判（如「云数据库为空」）。供全员联调与答辩使用。

---

## 一、总体架构（混合方案）

```text
┌─────────────────────────────────────────────────────────┐
│                    微信小程序端                          │
├─────────────────────────────────────────────────────────┤
│  业务数据（JSON）          │  图片文件（二进制）           │
│  Bmob 云数据库             │  微信云开发 · 云存储          │
│  · _User                   │  · items/   商品图           │
│  · Item                    │  · avatars/ 用户头像         │
│  · VerifyCode / Order…     │                              │
│  存：标题、价格、状态、      │  存：jpg 等文件               │
│      cloud:// 链接字符串   │  返回：cloud://fileID        │
└─────────────────────────────────────────────────────────┘
```

**原则**：数据库只存 **链接**（`cloud://…` 或历史 https），不存图片二进制。  
**不是**「把图存进 Bmob 表」或「把商品存进微信云数据库」。

## 二、各控制台分别管什么

| 控制台 | 菜单 | 本项目用途 | 为空是否正常 |
|--------|------|------------|--------------|
| **Bmob** | 云数据库 → `_User` | 用户、昵称、手机号、`avatarUrl` | 应有登录用户 |
| **Bmob** | 云数据库 → `Item` | 物品/跑腿发布记录 | 未发布前可空；发布后应有数据 |
| **Bmob** | 云数据库 → `VerifyCode` | 校园邮箱验证码 | 发验证码后有记录 |
| **Bmob** | 设置 → 应用配置 | 微信小程序 **AppID / AppSecret**（登录） | 必须填对 |
| **Bmob** | 应用密钥 | `SECRET_KEY`、`API 安全码` → 对应 `utils/bmob.js` | 与微信 Secret 不同 |
| **微信云开发** | **存储** | 真实图片文件 | 上传成功后应有 `items/`、`avatars/` |
| **微信云开发** | **数据库** | 本项目 **未使用** | **一直为空是正常的** |

## 三、配置文件一览（改哪里）

### 3.1 小程序工程

| 文件 | 配置项 | 说明 | 是否提交 Git |
|------|--------|------|--------------|
| `project.config.json` | `appid` | 团队正式小程序 ID（当前 `wx84eb83299b4116c0`） | 是 |
| `utils/cloudConfig.js` | `CLOUD_ENV_ID` | 云开发环境 ID，**全员相同** | 是（仅 ID，无密钥） |
| `utils/cloudConfig.js` | `CLOUD_STORAGE_ENABLED` | `true` 走云存储传图；`false` 走 Bmob 文件（需备案域名） | 是 |
| `utils/bmob.js` | `SECRET_KEY`、`API_SAFE_CODE` | Bmob 应用密钥，**不是**微信 AppSecret | 按团队约定（勿公开到网盘） |
| `app.json` | `tabBar.custom` | 自定义底部栏（首页 / 发布 / 我的） | 是 |

### 3.2 Bmob 控制台（网页，不写进小程序仓库）

| 配置 | 位置 | 内容 |
|------|------|------|
| 微信登录 | 设置 → **应用配置** → 微信小程序帐号服务 | AppID、AppSecret（与 `project.config.json` 一致） |
| 应用密钥 | 设置 → **应用密钥** | 复制到 `utils/bmob.js` 的 `SECRET_KEY` |
| API 安全码 | 设置 → **安全验证** | 复制到 `utils/bmob.js` 的 `API_SAFE_CODE` |
| 文件域名 | 设置 → **域名管理** | 仅当 `CLOUD_STORAGE_ENABLED = false` 时需要备案域名 |

### 3.3 微信云开发（开发者工具内）

| 配置 | 位置 | 说明 |
|------|------|------|
| 环境 ID | 云开发 → 设置 | 写入 `CLOUD_ENV_ID`（示例：`cloud1-d3gw967ood48ca1e4`） |
| 存储权限 | 云开发 → 存储 → 权限 | 建议「所有用户可读，仅创建者可写」；仅创建者可读时他人看图可能 403 |
| 本地调试 | 详情 → 本地设置 | 勾选 **不校验合法域名** |

## 四、字段与存储位置对照

### 4.1 `Item` 表（Bmob）

| 字段 | 存什么 | 示例 |
|------|--------|------|
| `title`、`price`、`status`… | 业务文本/数字 | 正常字符串 |
| `coverImage` | 封面图 **地址** | `cloud://cloud1-xxx.../items/xxx.jpg` |
| `images` | 多图 JSON 字符串 | `["cloud://...", "cloud://..."]` |

### 4.2 `_User` 表（Bmob）

| 字段 | 存什么 |
|------|--------|
| `nickName`、`phone`、`campus`… | 用户资料 |
| `avatarUrl` / `userPic` | 头像 **cloud:// 链接**（个人设置保存时写入） |

### 4.3 云存储目录

| 目录 | 来源页面 | 写入方式 |
|------|----------|----------|
| `items/` | 物品/跑腿发布 | `utils/cloudStorage.uploadImage(path, 'items')` |
| `avatars/` | 个人设置头像 | `utils/cloudStorage.uploadImage(path, 'avatars')` |

## 五、代码模块职责

| 模块 | 文件 | 作用 |
|------|------|------|
| Bmob 初始化 | `utils/bmob.js` | SDK 密钥 |
| 登录/用户 | `utils/auth.js` | 微信登录、更新 `_User` |
| 发布读写 | `utils/publish.js` | `saveItem` / `getItem`；上传走云或 Bmob |
| 云上传 | `utils/cloudStorage.js` | `wx.cloud.uploadFile` |
| 云图展示 | `utils/cloudImage.js` | `cloud://` → 临时 https，避免列表 403 |
| 云开关 | `utils/cloudConfig.js` | 环境 ID、是否启用云存储 |

## 六、常见问题（排错）

| 现象 | 原因 | 处理 |
|------|------|------|
| 云开发 **数据库** 无集合 | 未使用云数据库 | **忽略**，看 Bmob `Item` 与云 **存储** |
| Bmob 只有 `_User`、`VerifyCode` | 未发布过商品或未刷新 | 发布一条后刷新；或手动「添加表 `Item`」 |
| 上传报「绑定文件域名」 | 仍在用 `Bmob.File` | 保持 `CLOUD_STORAGE_ENABLED = true` |
| 列表有字无图、Console **403** | 云存储读权限 / 临时链 | 改存储权限；已用 `cloudImage.js` 解析 |
| 登录 40029 / 10007 | Bmob 与公众平台 AppID/Secret 不一致 | 在 Bmob **应用配置** 改微信项 |
| 头像上传失败 | 原逻辑走 Bmob 文件 | 已改云存储 `avatars/`，需点 **保存** 写入 `_User` |
| TabBar 编译错误 pagePath 重复 | 两个 Tab 指向同一页 | 已用 `pages/publishHub/publishHub` 占位 |

## 七、新成员 5 分钟检查清单

1. 克隆代码，打开 `secondhand`，AppID 与 `project.config.json` 一致。  
2. 确认 `utils/cloudConfig.js` 中 `CLOUD_ENV_ID` 与负责人一致。  
3. **工具 → 构建 npm**。  
4. **不校验合法域名**。  
5. 我的 → 登录 → 发布一条带图商品 → Bmob 看 `Item`、云开发看 **存储** `items/`。  
6. 个人设置 → 换头像 → 保存 → 云存储 `avatars/`、Bmob `_User.avatarUrl`。

## 八、与备案 / 小程序备案的关系

| 类型 | 是否必须（当前方案） |
|------|----------------------|
| Bmob **文件域名**备案 | **否**（图片走云存储） |
| 微信 **小程序备案** | 正式上线时需要；本地开发/体验版演示通常可先不做 |
| 域名备案 | 仅当改回 Bmob 传图或自建图床时需要 |
