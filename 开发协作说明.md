## 一、项目简介

「校园二手交易平台」微信小程序，面向在校学生提供二手物品、跑腿任务发布与浏览、交易管理等功能。后端使用 **Bmob 后端云**（BaaS），无自建服务器，无支付/资金链路。

---

## 二、技术栈

| 项目 | 说明 |
|------|------|
| 框架 | 微信小程序原生（WXML + WXSS + JS），ES6+ |
| 后端 | Bmob 后端云，SDK: `hydrogen-js-sdk`  |
| 登录 | `wx.login` → Bmob `loginWithWeapp` 一键注册/登录 |
| 校园认证 | 校园邮箱验证码（`.edu.cn`），Bmob 云函数 `sendVerifyEmail` |
| 单位 | rpx（响应式像素），适配 iPhone / Android |
| 工具 | 微信开发者工具（需构建 npm） |

## 六、开发环境搭建

### 6.1 准备工作（所有人必做）

1. 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 克隆项目，在微信开发者工具中导入项目目录
3. **工具 → 构建 npm**，确保 `miniprogram_npm` 生效

### 6.2 Bmob 数据库配置

#### 代码层面：bmob.js 已配置好，拉下来直接用

`utils/bmob.js` 里已经填好了 **Secret Key** 和 **API 安全码**。

**所有人 clone 代码后完全不需要修改 `bmob.js`。** 在本地运行小程序、注册账号、发布商品，数据都会自动存到同一个 Bmob 数据库里。

```js
// utils/bmob.js —— 已配置完毕，无需修改
const SECRET_KEY = '已填写'      // 团队共享，不要改
const API_SAFE_CODE = '已填写'   // 团队共享，不要改
```

#### 是否需要注册 Bmob 账号？

取决于分工角色：

**情况 A（不需要注册 Bmob 账号）：**
- 你只负责写**纯前端页面**（画 UI、写样式、页面布局）
- 或者你完全通过调用已封装好的工具函数（`utils/auth.js` 等）来读写数据，不需要去数据库后台查看原始数据
- **你可以零 Bmob 账号参与开发**——只要微信开发者工具能跑起来就行

**情况 B（建议注册 Bmob 账号）：**
- 你负责的模块涉及数据增删改查（如"商品发布"、"订单模块"、"用户系统"）
- 写完代码后你需要去数据库后台看一眼数据有没有存进去、字段对不对、关联是否正确
- **注册步骤：**
  1. 去 [Bmob 官网](https://www.bmobapp.com/) 注册一个账号（`bmob.cn` 若无法访问请用 `bmobapp.com`）
  2. 把注册邮箱发给项目负责人
  3. 负责人操作：登录 Bmob → 进入应用 → 左侧菜单栏最下方 **"应用设置" → "协作管理"** → 添加你的 Bmob 账号
  4. 添加后，你用自己账号登录 Bmob，就能看到团队共用的数据库表了

### 6.3 关于微信小程序 AppID / AppSecret

- **代码里只改** `project.config.json` 的 `appid`（当前团队正式号：`wx84eb83299b4116c0`）。全员拉代码后一致即可。存储与云开发配置见 [docs/数据存储与配置说明.md](./docs/数据存储与配置说明.md)。
- **微信 AppSecret 不要写进任何 `.js` / `.json` 仓库文件**（会泄露）。只填在 **Bmob 控制台**（见下）。
- 登录用的 `utils/bmob.js` 里是 **Bmob 的 Secret Key + API 安全码**，与微信 AppSecret 不是同一个东西；换小程序 AppID 时 **一般不用改** `bmob.js`，除非换了 Bmob 应用。

**Bmob 控制台（负责人，登录必做）：**  
应用 → **设置** → **应用密钥 / 微信小程设置**（名称以控制台为准）→ 填写与公众平台一致的 **AppID**、**AppSecret** → 保存。

如果你需要在小程序后台（mp.weixin.qq.com）进行配置（如添加请求域名白名单），需要让负责人把你添加为**小程序开发者**：
1. 负责人登录 [微信公众平台](https://mp.weixin.qq.com/) → 成员管理 → 添加开发者
2. 你用自己的微信扫码绑定即可

### 6.4 本地调试

- 开发时勾选：详情 → 本地设置 → **不校验合法域名、web-view、TLS 版本**

## 七、开发规范

### 7.1 语法

```js
// 全部 ES6+
const Bmob = require('../../utils/bmob.js')
const r = await Bmob.Query('Item').get(id)      // async/await
const { title, price } = row                      // 解构
const msg = `价格：${price}`                      // 模板字符串
```

### 7.2 页面结构

每个页面四个文件，命名与文件夹名一致：

```
pages/xxx/xxx.js      # Page({...})
pages/xxx/xxx.json    # { "usingComponents": {} }
pages/xxx/xxx.wxml    # 模板
pages/xxx/xxx.wxss    # 样式
```

### 7.3 登录保护

需要登录的页面，在 `onLoad` 最前面检查：

```js
onLoad() {
  if (!auth.checkLoginStatus()) {
    wx.redirectTo({ url: '/pages/login/login' })
    return
  }
  // 正常逻辑
}
```

或调用 `getApp().checkAndLogin()`（会先弹登录页并返回 boolean）。

### 7.4 数据查询

```js
const Bmob = require('../../utils/bmob.js')

// 查询
const q = Bmob.Query('Item')
q.equalTo('status', '==', 'ON_SALE')
q.order('-createdAt')
q.limit(10).skip(page * 10)
const list = await q.find()

// Pointer 关联查询
q.equalTo('sellerId', '==', Bmob.Pointer('_User').set(userId))
q.include('sellerId')  // 联查卖家信息

// 更新
const row = await Bmob.Query('Item').get(objectId)
row.set('status', 'OFFLINE')
await row.save()

// 删除
await Bmob.Query('Cart').destroy(objectId)
```

### 7.5 列表页模板

所有列表页遵循统一模式：

```js
Page({
  data: {
    list: [],        // 数据
    page: 0,         // 当前页
    loading: true,   // 首次加载
    loadingMore: false,
    refreshing: false,
    hasMore: true
  },
  async loadData(reset) { /* 分页查询，处理三种状态 */ },
  onRefresh() { /* 下拉刷新 */ },
  onLoadMore() { /* 上拉加载 */ }

```

### 7.6 样式

- 使用 `rpx` 单位
- 主色 `#2d5a8e`，辅色 `#c45b3e`，背景 `#f5f5f5`
- 全局工具类在 `app.wxss`：`.btn-primary` `.btn-outline` `.btn-danger` `.card` `.tag-*` `.flex-row` `.text-muted` 等
- 页面级样式写在对应 `.wxss` 中，避免滥用内联样式
- WXML 列表渲染必须加 `wx:key`

### 7.7 Bmob Date 类型

查询日期比较时需使用 `{ __type: 'Date', iso: '...' }` 格式：

```js
const dateGtNow = { __type: 'Date', iso: new Date().toISOString() }
q.equalTo('expireAt', '>', dateGtNow)
```

## 八、关键约定

1. **无资金链路**：不接入微信支付，不下单结算，Cart 表本质是「意向收藏」
2. **服务端权威**：Bmob 数据库为准，本地缓存仅用于加速首屏展示，进入关键页面时需从 Bmob 刷新最新数据
3. **软删除**：商品删除使用 `status = 'DELETED_SOFT'`，不物理删除
4. **商品状态枚举**：`ON_SALE` / `IN_TRADING` / `SOLD_OUT` / `OFFLINE` / `DELETED_SOFT`
5. **订单状态枚举**：`PENDING_CONFIRM` / `IN_TRADING` / `COMPLETED` / `CANCELLED`
6. **用户状态枚举**：`active` / `frozen` / `disabled`
7. **页面间通信**：通过 URL 参数或 `getApp().globalData`，不用事件总线
8. **新增页面后**：需在 `app.json` → `pages` 数组中注册（微信小程序按数组顺序编译首页）
